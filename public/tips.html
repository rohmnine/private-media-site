使用说明

Guide

了解如何维护这个像素垃圾博物馆，并为后续部署做准备。

  1. 推荐通过 npm start 启动本地 Node.js 服务，再用 http://127.0.0.1:8080 或局域网地址访问。直接双击 HTML 只能使用静态预览。
  2. 影像碎片、视觉残片、声音碎片页面支持入馆登记并持久保存到后端配置的媒体目录；默认分别是 assets/video、assets/photos、assets/audio。
  3. 可以在 .env 中配置 MEDIA_ROOT、DATA_ROOT、TRASH_ROOT、THUMB_ROOT，把藏品、数据、废弃区和缩略图缓存迁移到数据盘或 NAS。
  4. 每件媒体藏品会生成 museumId 藏品编号，并带有 collectionType：电子垃圾 / 现实垃圾 / 生活日志。
  5. item.html 展示藏品详情，timeline.html 按日期混合展示照片、视频、音频和日志。
  6. 删除藏品会先移入废弃区，可在废弃区页面恢复、彻底销毁或清空；重要操作会自动写入系统日志。
  7. 入馆登记会校验扩展名、MIME 类型和常见文件头；不匹配的文件会被跳过，登记失败时会清理临时文件。
